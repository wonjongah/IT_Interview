##### 동기화

<u>네트워크 통신 프로토콜의 역할 중 송신 측에서 전송한 데이터의 각 비트를 수신 측에서 정확하게 수신할 수 있도록 하는 기법</u>



##### 동기식 전송

- 송수신기가 동일한 클록을 사용해 데이터를 송수신
- 미리 정해진 수만큼 <u>문자열을 한 묶음으로 만들어 일시에 전송</u>하는 방법
- 파일 업로드, 파일 다운로드
- 수신측에서 비트 계산을 해야 하고 문자를 조립하는데 별도의 기억장치가 필요
- 비트지향 동기화 방법, 문자지향 동기화 방법



##### 비트지향 동기화 방식

- <u>전송하는 모든 데이터 단위를 일련의 비트로 처리. 동기화에 필요한 데이터까지 비트로 표현</u>
- 데이터 블록의 처음과 끝에 8비트의 플래그 비트(01001100) 표시
- 플래그 등 패턴을 구별하려고 0비트 삽입
- 시작플래그/주소/제어필드/데이터블록/FCS/종료플래그



##### 문자지향 동기화 기법

- <u>모든 데이터의 단위를 문자 단위로 처리함으로써 동기화에 필요한 데이터까지 문자로 표현</u>
- SYN : 동기맞춤
- STX : 문장의 시작
- ETX : 문장의 끝
- SYN/SYN/STX/DATA/ETX/BCC



##### 비동기식 전송

- 송수신기가 별도의 독립적인 클록을 사용하면서도 데이터를 송수신
- <u>한 번에 한 문자씩 전송함으로써</u> 수신기가 새로운 문자의 시작점에서 재동기화
- START 비트와 STOP 비트 이용
- 패리티 비트를 통한 오류 검출



##### IPv6

- IPv6 : IPv4의 주소 부족 문제를 해결하기 위해 개발되었다.
- 16비트씩 8부분, 총 128비트로 구성되어, 각 부분을 16진수로 표현하고 콜론으로 구분되어 있다.
- 인증 및 보안 기능을 확장헤더에서 기본으로 제공한다.



##### 디지털 변조 방식

컴퓨터에서 발생하는 디지털 신호를 아날로그 신호로 바꿔주는 기술

<u>디지털 신호 -> 아날로그 신호</u>

| 구분                  | 설명                                                         |
| --------------------- | ------------------------------------------------------------ |
| 진폭 편이 변조(ASK)   | 반송파의 진폭을 2 또는 4개로 정해놓고 데이터가 1 또는 0으로 변함에 따라 미리 약속된 진폭의 반송파를 수신 측에 전송하는 방식 |
| 주파수 편이 변조(FSK) | 반송파의 주파수를 높은 주파수와 낮은 주파수로 정해 놓고 데이터가 0이면 낮은 주파수, 1이면 높은 주파수를 전송하는 방식, 전이중 방식 |
| 위상 편이 변조(PSK)   | 송신 측에서 반송파의 위상을 2, 4, 8 등분 등으로 나누어 각각 다른 위상에 0 또는 1을 할당하거나, 2나 3비트로 한꺼번에 할당하여 수신 측에 전송하는 방식 |
| 펄스 부호 변조(PCM)   | 아날로그 신호를 디지털 신호로 변환하여 전송하는 방식. 표본화, 양자화, 부호화해서 변조 |



##### 펄스 부호 변조(PCM) 과정

- 아날로그 신호를 디지털 신호로 변환하여 전송하는 방식
- <u>표본화 -> 양자화 -> 부호화</u>로 송신하고 역으로 복호화하여 수신한다.

| 전송과정 | 설명                                                         |
| -------- | ------------------------------------------------------------ |
| 표본화   | 연속적으로 변화하는 값을 일정한 시간적 간격으로 나누어 표본을 추출한다. PCM 아날로그 음성신호를 디지털화하기 위한 사전작업으로 샘플링 타임 간격으로 나누어 추출하는 과정 |
| 양자화   | 연속적인 아날로그 값을 이산적인 디지털 값으로 바꾸는 A/D 변환. 표본화된 펄스의 크기를 부호화하기 위한 값으로 바꾸어주는 과정 |
| 부호화   | 각 데이터 정보 하나하나에 할당되는 2진 표현으로 바꾸는 과정  |



##### 다중화 

<u>고속의 전송로 하나에 데이터 신호 여러 개를 중복시켜 고속 신호 하나를 만들어 전송하는 방식</u>

| 종류 | 설명                                                         |
| ---- | ------------------------------------------------------------ |
| TDM  | 주어진 시간대역을 여러 타임 슬롯으로 분할하여 각 채널에 배당함으로써 많은 채널이 하나의 전송회선을 공유하는 방식. 동기/비동기 방식 |
| FDM  | 대역이 제한된 전송회선의 주파수 대역을 여러 개로 분할하여 각 채널에 배당함으로써 많은 채널이 하나의 전송회선을 공유하는 방식 |
| CDM  | 여러 신호에 각기 다른 코드 시퀀스를 부여한 후 하나의 채널로 다중화하여 전송하는 방식. spread spectrum의 원리에 기초를 둔 방식. FDM + TDM의 혼합방식 |
| SDM  | 공간적으로 분리된 채널을 단위로 다중화하여 전송. 무선은 용량을 증대. 스마트안테나 또는 위상배열안테나 이용 서비스 영역을 여러 개의 빔으로 커버하는 다중 빔화 |
| DWDM | 광 파장을 실어 전송하여 전송용량과 채널의 수를 늘린 기술. 동시 전송 |

##### <u>ARG</u>(Auto Repeat Request) 방식

<u>오류가 발생하면 수신 측은 송신 측에 오류가 발생한 사실을 알리고 송신 측에 오류가 발생한 프레임을 전송할 것을 요구하는 기법</u>

- <u>Stop-and-Wait ARQ</u>
  - 에러가 없을 경우 송신측에게 ACK를 전송
  - 에러가 있는 경우 NAK를 전송하여 재전송 유도 : 하나만 보내고 하나 확인 후 다시 전송

- <u>Sliding Window ARQ</u>
  - <u>Go-Back-N</u>
    - 1~5 수신했는데 3번이 에러면 3번부터 다시 전송
    - 오류난 곳부터 다시 전송
  - <u>Selective-Repeat</u>
    - 에러가 발생한 프레임만 재전송
    - 1~5 보냈는데 3번 오류면 3번만 다시 전송

##### IP 주소 체계

<u>인터넷에 연결된 모든 컴퓨터를 네트워크 상에 구별하는 고유의 주소체계</u>

| 주소 종류                    | 설명                                                         |
| ---------------------------- | ------------------------------------------------------------ |
| <u>클래스 타입</u>           | IP 주소를 효율적으로 배정. A,B,C,D,E 클래스. D : 멀티캐스트, E : 예비용. D클래스는 Netid와 Hostid 없음 |
| <u>네트워크 주소(Net ID)</u> | 전체 네트워크를 좀 더 작은 네트워크로 분할하여 <u>각 호스트가 속한 네트워크를 대표.</u> 8비트, 16비트, 24비트 |
| <u>호스트 주소(Host ID)</u>  | 네트워크 주소로 표현하는 네트워크 내부에서 <u>각 호스트의 주소를 표현하는 역할</u>. 전체 32비트에서 네트워크 주소를 제외한 나머지에 해당 |



##### IP 주소

Version 4의 IP 주소는 8비트 크기의 필드 네 개를 모아서 구성한 32비트 논리주소



##### 3Way-HandShaking(3방향 연결설정)

<u>TCP 프로토콜은 단말기와 단말기 간의 세션을 설정할 때, 3방향 연결설정을 통합하여 신뢰성 있는 정보 전송을 제공하는 기법</u>이다.



##### 3Way-HandShaking(3방향 연결 설정) 절차

| 절차          | 설명                                                         | FLAG 설정 |
| ------------- | ------------------------------------------------------------ | --------- |
| 연결설정요구  | A 프로세스가 TCP 헤더의 SYN 플래그를 지정한 세그먼트를 전송  | SYN       |
| 연결 수락     | 연결설정 요구를 받은 B프로세스가 연결을 수락하려면 긍정응답을 해야 한다. SYN과 ACK 플래그를 지정해 연결에 대한 긍정 응답을 표시. SYN 플래그가 지정한 세그먼트에는 전송 데이터가 포함되지 않지만 순서 번호는 1 증가. SYN 세그먼트의 순서번호 10에 1을 더한 11번을 Ackowledgement Number 필드에 지정하여 회신. Ackowledgement Number 값을 유효하게 하기 위해 ACK 플래그 지정. B 프로세스의 순서 번호 50은 임의로 지정된 값 | SYN, ACK  |
| 연결수락 확인 | B 프로세스가 전송한 연결 수락 세그먼트가 도착했음을 알림. 전송할 데이터가 있는 경우 바로 데이터 전송 | ACK       |

| -------- | ------------------------------------------------------------ |
| TCP      | <u>전송제어 프로토콜</u>, <u>네트워크의 정보전달을 통제</u>하는 프로토콜. 인터넷을 이루는 핵심 프로토콜의 하나로서 IETF의 RFC793에 기술 |
| UDP      | 신뢰성 낮은 데이터그램 프로토콜로서 완전성을 보증하지 않으나, 응용이나 네트워크 환경에서 가상회선을 굳이 확립할 필요가 없어서 유연하고 효율적인 데이터. <u>전송에 사용되는 프로토콜</u> |
| RTP      | 인터넷상에서 다수가 종단간에 <u>실시간으로 비디오나 오디오 패킷을 전송하기 위해 표준화된 프로토콜</u> |
| SCTP     | <u>멀티미디어 통신</u>을 위해 UDP와 TCP의 일부 장점을 결합하여 설계되었으며, 고도의 신뢰 전송을 제공하는 전송계층 프로토콜 |

- 네트워크 계층

| 프로토콜     | 설명                                                         |
| ------------ | ------------------------------------------------------------ |
| <u>ARP</u>   | TCP/IP 네트워크에서 ARP 프로토콜은 <u>IP주소를 물리 주소로 변환 프로토콜</u>(IP->물리) |
| <u>RARP</u>  | 물리주소인 <u>MAC 주소를 IP 주소로 변환시켜주는 역주소 변환 프로토콜</u>(물리->IP) |
| <u>IPSec</u> | <u>네트워크 계층에서 IP패킷 단위로 인증 및 암호화를 하는 프로토콜</u> |
| <u>ICMP</u>  | TCP/IP에서 <u>IP패킷을 처리할 때 발생되는 문제를 알리거나, 진단 등과 같이 IP 계층에서 필요한 기타 기능들을 수행하기 위해 사용되는 프로토콜</u> |

##### 슬라이딩 윈도우 기법

윈도우 크기 단위로 바이트를 송신하고, 수신 측에서 수신한 바이트에 대한 확인/응답 신호를 전송하면, <u>한꺼번에 윈도우 크기만큼 버퍼가 미끄러지듯이 이동한다.</u> 호스트는 각 연결 세션마다 하나의 윈도우를 이용한다.



##### Stop and Waiting 기법

수신자가 에러 체크 후 에러가 있으면 에러가 난 데이터부터 하나씩 다시 요청해서 받는 방식.



##### TCP 혼잡제어 기법

<u>TCP 프로토콜에서 네트워크가 혼잡하다고 판단될 때 송신률을 감속시킨다.</u> 혼잡제어는 수용할 수 있는 이상의 데이터 전송이 이루어질 때 발생하는 패킷 전송 지연 혹은 패킷 손실을 막는 목적으로 사용하며 대표적인 기법은 <u>슬로우 스타트</u> 기법이다.



##### 슬로우 스타트 기법

<u>미리 정해진 임계값에 도달할 때까지 윈도우의 크기를 2배씩 증가</u>시킨다. <u>전송 시마다 크기가 두 배씩 증가하기 때문에 전송되는 데이터의 크기는 지수 함수적으로 증가한다.</u> <u>전송되는 데이터의 크기가 임계값에 도달하면 윈도우의 크기를 반으로 줄이는 혼잡회피 단계로 넘어가는 기법이다.</u>



##### 혼잡 회피 기법

전송되는 데이터 크기가 임계값에 도달하면 윈도우의 크기를 1로 줄인다. 동시에 임계값을 패킷 손실이 발생했을 때 <u>윈도우 크기의 반으로 줄어든다.</u>



##### UPD(User Datagram Protocol)

- TCP/IP 프로토콜 중 트랜스포트 계층의 통신 프로토콜의 하나로 신뢰성이 낮은 프로토콜로써 완전성을 보증하지 않으나, <u>가상회선을 확립할 필요가 없고 유연하며 효율적 응용의 데이터 전송에 사용한다.</u>
- UDP는 비연결성이고 신뢰성이 없으며, 순서화되지 않은 Datagram 서비스를 제공하고 실시간 응용 및 멀티캐스팅이 가능하다.



##### UDP의 주요 특징

비연결성, 비신뢰성, 순서화되지 않은 Datagram 서비스, 실시간 응용 및 멀티캐스팅 가능, 단순한 헤더, 데이터 전송 단위, 최대 데이터 크기



##### TCP VS UDP

| 구분        | TCP                                  | UDP                                                    |
| ----------- | ------------------------------------ | ------------------------------------------------------ |
| 정의        | 3, 4 계층의 연결지향적 통신 프로토콜 | 빠른 응답을 요구하는 비연결형 설정을 제공하는 프로토콜 |
| 데이터 순서 | 보내는 순서 유지                     | 순서 유지하지 않음                                     |
| 데이터 중복 | 데이터 중복, 손실 없음               | 데이터 중복, 손실 가능                                 |
| 에러 제어   | 에러 검사 후 에러시 재전송           | 에러 검사 후 에러시 재전송하지 않음                    |
| 흐름제어    | 슬라이딩 윈도우                      | 흐름제어 없음                                          |
| 종류        | telent, ftp, smtp                    | SNMP, TFTP                                             |
| 장점        | <u>신뢰성 있는 경로 확립</u>         | <u>빠른 전송</u>                                       |
| 단점        | 전송에러 감시 부하                   | 안전 전송 여부 확인 못함                               |



##### 무선 LAN

LAN 등의 물리적 케이블 없이, 무선주파수 등을 이용하여, 노드 간 데이터 송수신을 지원하는 네트워킹 기술로 <u>무선 LAN 표준</u>은 <u>IEEE 802.11</u>이다.

##### NAT(Network Access Translation)

<u>외부 네트워크</u>에 알려진 것<u>과 다른 IP 주소를 사용하는 내부 네트워크에서 IP 주소를 변환하는 기술</u>



##### <u>DHCP</u>(Dynamic Host Configuration Protocol)

네트워크 관리자가 일정한 시간 동안만 유효하도록 하는 <u>임대 개념의 IP 주소를 할당하고 관리할 수 있도록 하는 프로토콜</u>

| 메커니즘                    | 설명                                                         |
| --------------------------- | ------------------------------------------------------------ |
| <u>Automatic Allocation</u> | Client에게 영구적인 IP를 자동으로 할당하는 기법              |
| <u>Dynamic Allocation</u>   | Client에게 <u>한정된 시간 동안 IP를 할당</u>하는 기법. IP 재사용까지 자동으로 이루어짐 |
| <u>Manual Allocation</u>    | Client의 IP Address를 Network 관리자가 <u>수동으로 할당</u>. DHCP는 할당된 주소를 재할당하는 기법 |



##### <u>CDN</u>(Contents Delivery Network)

컨텐츠 제공업자의 웹서버에 집중되어 있는 용량이 크고 사용자 요구가 잦은 컨텐츠를 통신사가 데이터센터에 설치한 CDN 서버에 미리 저장하여 최적의 경로로 사용자에게 컨텐츠를 전달하는 기술



##### CDN의 주요 기술

| 특징                         | 설명                                                         |
| ---------------------------- | ------------------------------------------------------------ |
| Cashing 기술                 | 자주 찾는 페이지를 컴퓨터에 복사해 저장한 후 사용자가 찾으면 저장된 정보를 전송 |
| Global Server Load Balancing | CDN 서비스를 제공하기 위해 인터넷 망에 분산 배치되어 있는 많은 캐시 서버 중에 이용자에게 최상의 서비스를 제공할 수 있는 캐시 서버를 선정하여 연결 |
| Load Balancing 기술          | 트래픽 분산을 통해 웹 고객에 대한 서비스 성능 향상. 장애가 난 서버를 제외 후 나머지 서버들에게 트래픽 할당 |
| Streaming 기술               | 대용량의 멀티미디어 데이터를 다운받지 않고 즉시 재생         |
| 컨텐츠 배포                  | 각 지역에 분산되어 있는 서버에 동일한 컨텐츠를 정확히 배포   |
| 동기화 기술                  | 컨텐츠 변경 시 즉각 반영해 동일한 컨텐츠로 서비스            |
| 분산 기술                    | 분산 데이터 관리 및 분산 병렬 처리 기술. Grid Computing, Virtualization 기술 적용. |



##### SDN(SW Defined NW)

- 소프트웨어 기술(오픈플로우)을 기반으로 네트워크 장비의 <u>패킷 포워딩 기능과 컨트롤러 기능을 표준 인터페이스로 분리하여 네트워크 개방성을 제공</u>하는 기술
- 이기종 벤더 네트워크 장비의 통합, 소프트웨어 기반으로 라우팅 정의
- 다양한 네트워크 장비 벤더사에 표준 API 제공



##### 오픈플로우

<u>네트워크 장비를 제어하기 위한 기능을 물리적 네트워크와 분리하여 데이터 전달과 상호작용하기 위한 오픈소스 프로토콜</u>



##### 릴리즈 노트

- 릴리즈 노트는 <u>어플리케이션 최종 사용자에게 전달되는 패키지 소프트웨어 제품의 릴리즈 정보에 대한 문서</u>이다.
- 릴리즈 노트의 헤더에는 릴리즈 노트 이름, 제품 이름, 버전 번호, 릴리즈 일자, 노트 버전 등을 기록하고 기타 변경 내용을 기록한다.
- 릴리즈 노트는 보통 패키징 소프트웨어 메뉴얼 또는 인터넷을 통해 공개한다.

##### 랜섬웨어와 랜섬웹

##### SLA(Service Level Agreement)

IT 아웃소싱 서비스를 제공하는 업체와 사용하는 업체 간의 서비스에 대한 책임과 의무사항을 기술한 <u>협약서</u>



##### SLA의 운영절차(정도 협측 보통)

서비스 정의 -> 서비스 도출 -> 서비스 협약 -> 서비스 측정 -> 서비스 보고 -> 서비스 통제

1. 준비 단계(정의/도출) : 서비스 요구 문서화, 초기 KPI 정의
2. 협약 및 측정 : 목표치, 역할/책임 및 절차 정의, 협상에 의한 협정 체결
3. 운영 및 평가 : 목표 대비 측정치 비교 등의 성과 보고/개선안 및 수준 개선



##### SCTP(Streaming Control Transmission Protocol)

- <u>멀티미디어 통신</u>을 위해 <u>UDP와 TCP의 일부 장점을 결합해 설계</u>
- 전송경로 장애를 대비한 대체경로 확보 기능, Multi-Streaming 기능 등이 포함된 고도의 신뢰전송을 제공하는 전송계층 프로토콜



##### SCTP의 기능

| 주요기능     | 설명                                                         |
| ------------ | ------------------------------------------------------------ |
| 멀티호밍     | SCTP 세션이 여러 개의 IP주소 동시 사용, 장애 발생 시 대체 경로(대체 IP주소) |
| 멀티스트리밍 | 하나의 세션을 통해 다양한 종류의 응용 데이터를 보냄. 세션 초기화 단계에서 송신자는 자신이 전송할 스트림의 개수를 수신자에게 통보하며 전송단계에서 각 스트림 별로 독립적인 순서화 가능 제공. 데이터 복구 및 재전송 과정 또한 스트림 ID별로 수행되며, TCP에서 문제되었던 HOL블록킹 문제 해결 |
| 세션 초기화  | TCP 3-way handshake와 달리 4-way handshake 사용. 4-way handshake 초기화 절차는 TCP SYN 공격 방지 |
| 데이터 전송  | 데이터 패킷에 TSN 및 SSN 값이 부여. TSN은 세션에 대한 흐름제어 및 오류 복구에 사용. SSN은 수신 측 SCTP에서 스티림 별 순서화에 사용 |
| 세션 종료    | TCP에서는 종료 시 4단계 절차였으나, SCTP에서는 3단계 절차 사용하여 TCP의 Half-Open 문제 해결 |



##### RAID(Redundant Array of <u>Independent</u> Disks)

- 처리속도, 데이터 보호를 목적으로 여러 개의 물리적인 디스크를 <u>중복성을 가진 하나의 논리적인 디스크 어레이로 변환하는 기술</u>
- <u>스트라이핑 기법</u>을 통해 데이터를 여러 조각으로 나누어 여러 디스크로 분산 저장하고 동시 접근을 통해 처리속도 향상 가능
- <u>중복 기법</u>을 통해 디스크에 저장된 일부 데이터가 손실된 경우에도 복구할 수 있도록 데이터를 중복하여 저장



##### 하이퍼바이저

- 하나의 서버나 컴퓨터에서 자원의 효율적 사용을 목적으로 동시에 다수의 운영체제를 구동시킬 수 있는 하드웨어와 운영체제 사이의 <u>얇은 소프트웨어 가상화 기술</u>
- 대표적인 하이퍼바이저로는 VM Ware, Hiper-V, Xen, KVM 등이 있다.



##### 도커

- <u>하이퍼바이저 없이 리눅스 컨테이너 기술을 바탕으로 소프트웨어의 배포를 자동화하는 기술</u>
- 서버의 운영체제는 공유하고 어플리케이션이나 프로세스는 격리하여 샌드박스 처리

##### Node.js

- <u>구글에서 만든 오픈소스 가상머신 형식의 자바 스크립트 V8 엔진 위에서 비동기 I/O로 서버 사이드 애플리케이션 개발에 사용되는 소프트웨어 프레임워크</u>
- Direct I/O 금지, Low-level, Built in, 플랫폼 독립적, 무상 라이센스



##### Node.js의 특징

- <u>비동기</u> : 비동기 I/O를 구현하여 고속의 서버 기능 구현 가능
- <u>싱글쓰레드</u> : 멀티쓰레드 모델을 배제하고 싱글쓰레드 모델로 쓰레드 레벨의 Context Switching의 자원을 효율적으로 사용
- <u>Event Loop</u> : Callback 함수를 통한 Event Loop로 사용자의 요구사항을 처리



##### Node.js 구성요소

1. V8 : 오픈소스 JLT 가상 머신 형식의 자바 스크립트 엔진
2. Thread Pool : 비동기 I/O 라이브러리로 구성, Libeio
3. Event Loop : 다양한 기능을 가진 고성능 이벤트 루프 라이브러리, Libev
4. Node binding : C언어와 C++로 구성된 시스템 바인딩 레이어, 소켓, http
5. Node standard library : 자바스크립트 기본 라이브러리. 노드 바인딩과 연결

##### 다크 데이터(Dark data)

- 기업 내에 저장은 되어 있으나 분석되지 않은 데이터
- 현실적으로 기업 내에서 이렇게 분석되지 않은 데이터가 많으므로 이러한 데이터를분석할 가치가 있는지 판단하는 것이 중요하다.
- 분석이 수행되기 전에는 가치를 판단할 수 없는데, 분석하려면 인력, 비용 시간이 필요하므로 다크 데이터를 분석하기 위해서는 우선 데이터의 가치가 내부적으로 파악되어야 한다.



##### 빅데이터에서 데이터의 의미

빅데이터는 데이터의 크기보다 <u>데이터가 가지고 있는 가치</u>에 더 초점을 두어야 하며, 데이터의 가치는 데이터의 분석결과를 얼마나 잘 활용할 수 있는가로 평가된다.



##### 데이터의 유형

| 유형          | 설명                                                         |
| ------------- | ------------------------------------------------------------ |
| 스몰 데이터   | 빅데이터와 구분되는 이전의 데이터, 전통적인 데이터           |
| 다크 데이터   | 기업 내에 저장은 되어 있으나 분석되지 않은 데이터            |
| 패스트 데이터 | 실시간으로 처리하고 분석하기 위한 초대용량 데이터            |
| 스마트 데이터 | 빅데이터에 비해 다소 용량은 적지만 바로 분석할 수 있는 데이터. 품질이 보장되고 빠르게 분석 가능한 데이터 |
